SOC Response Playbooks

Incident response procedures for each threat category

PII Exposure Response
SSN, email, and personal identifier exposure
PII T1005 SLA: 15 min acknowledge
▼
Immediate Response (0-15 mins) 1. Acknowledge incident in Data-Aegis incident log
2. Identify source record, region, and accessor account
3. Verify masking was applied — confirm [REDACTED] tokens in output
4. Identify the data subject and notify data owner
Containment 1. Suspend accessor account if malicious behavior suspected
2. Review bulk access patterns — threshold is 5 records per session
3. Preserve all evidence — do not modify original logs
Escalation Criteria More than 10 records exposed, external contractor was the accessor, access occurred outside business hours, or same user flagged in multiple regions
GDPR Note 72-hour breach notification required if EU data subjects are affected
Credential Leak Response
Exposed AWS keys, passwords, and access tokens
Credentials T1552 SLA: 5 min — critical
▼
Immediate Response — Treat as Critical 1. Assume key is already compromised — rotate before investigating
2. Deactivate the exposed key in AWS IAM console immediately
3. Check CloudTrail for any usage of the key between exposure and rotation
4. Generate new credentials for the affected service account
Investigation 1. What permissions did this key have?
2. Was it used from any unknown IP addresses?
3. Check for lateral movement — did the key access unexpected resources?
Escalation Criteria Key has AdministratorAccess, evidence of unauthorized usage before rotation, or multiple keys exposed in same session
Healthcare Data Response
HIPAA breach response for patient records and diagnosis codes
Healthcare T1530 SLA: 15 min + legal
▼
Immediate Response (0-15 mins) 1. Notify Privacy Officer immediately — HIPAA mandates this
2. Identify which patient record was exposed and which account accessed it
3. Preserve all evidence — the Data-Aegis audit trail is your primary evidence
4. Restrict access to the affected healthcare data bucket
HIPAA Breach Assessment 1. Nature and extent of PHI involved
2. Who accessed or could have accessed the PHI
3. Whether PHI was actually acquired or only viewed
4. Extent to which risk has been mitigated
Notification Requirements Individual notification within 60 days. HHS notification if 500+ individuals affected. Media notification if 500+ state residents affected.
IAM Violation Response
Privilege escalation and unauthorized policy changes
IAM T1078 SLA: 15 min + revert
▼
Immediate Response (0-15 mins) 1. Identify what permissions were granted and to which account
2. Check for a valid change ticket — no ticket means treat as unauthorized
3. Revert the policy change immediately if no authorization found
4. Suspend the actor account if behavior looks malicious
Investigation 1. Review all actions by this account in the last 24 hours via CloudTrail
2. Check for persistence — were new IAM users or access keys created?
3. Did the escalated account access any sensitive resources after the change?
Escalation Criteria AdministratorAccess was granted, evidence of data exfiltration following the change, or an external contractor account made the modification

Detection Rules

Sigma rules and SQL queries for SIEM integration

Sigma AWS Credential Exposure Detection

title: Exposed AWS IAM Access Key in CloudTrail
id: 4a3b2c1d-ef12-3456-a7b8-90abcdef1234
status: experimental
description: Detects AKIA credential strings in log contents
logsource:
    product: aws
    service: cloudtrail
detection:
    selection:
        message|contains: 'AKIA'
    condition: selection
level: high
tags:
    - attack.credential_access
    - attack.t1552

SQL Privilege Escalation Detection

SELECT
    eventTime,
    userIdentity.arn        AS actor_arn,
    requestParameters.policyArn AS attached_policy,
    awsRegion
FROM aws_cloudtrail_logs
WHERE
    eventName IN ('AttachUserPolicy', 'PutRolePolicy')
    AND requestParameters.policyArn LIKE '%FullAccess%'
    AND eventTime >= DATE_SUB(NOW(), INTERVAL 1 HOUR)

ORDER BY eventTime DESC;
